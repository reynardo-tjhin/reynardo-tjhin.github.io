# About

The aim of this project is to get my hands dirty with software development. I focused more on Artificial Intelligence courses at university and missed out on a lot of software development. Therefore, I want to use this opportunity to learn software development from the ground up, all the way to deployment in the real world.

Since I have lots of personal files, I want to create a media server where I can store files and retrieve them, similar to cloud storage services such as Google Drive, Apple's iCloud, etc. Instead of using existing projects like Proxmox, I want to create my own project to understand how the web (or software) works. Through this project, I expect to learn about databases, backend, frontend, the hardware used to serve media files (and the software like Nginx), the deployment pipelines, data migration (when there's a change in the database) and so much more.

The first machine I used was the Beelink SER5 PRO Mini PC with the following:

- CPU: Ryzen 7 5850U 8C/16T(up to 4.4 GHz)
- Memory: 32GB DDR4
- Storage: 1TB PCIe3.0 SSD

The price I paid was AUD 519.00, which was fair. The pros of this machine are that it has a relatively strong CPU with 8 cores and 16 threads, it is a very quiet machine, has a decent amount of DDR4 RAM and a decent amount of storage capacity. However, the downside is that it is not meant to be a media storage machine. It lacks storage expandability and storage redundancy.

Therefore, like the homelabbers, I bought a used machine that supports storage expandability and storage redundancy. It is an old enterprise HP Z440 machine. It boasts the Intel Xeon E5-2630 v3 @ 2.2GHz with whopping 64GB of Enterprise DDR4 memory 🥳. I documented the steps I took to set up the machine on the next page.
